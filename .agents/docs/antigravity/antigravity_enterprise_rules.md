# 🚀 Ультимативный Профиль Enterprise Fullstack ИИ-Разработчика

В этом документе в финальном виде сформулирован **золотой стандарт Enterprise-разработки**. Теперь Antigravity — это автономная, отказоустойчивая IT-компания с Коллегиальным Мозгом, генеративным UI-слоем и жесткими рамками безопасности.

---

## 🏛️ 1. Коллегиальный Агент: Совет Директоров (The Architectural Council)

Чтобы избежать "галлюцинаций" Оркестратора, внедрен механизм независимого Коллегиального Ревью.

### 🛑 Защита от бесконечного совещания (Deadlock Prevention)
Главная опасность ИИ-коллегий — это "совещания ради совещаний" и бесконечные круги споров. Чтобы этого избежать, внедрено правило **Strict One-Shot Review (Строгий Единичный Опрос)**:
1. **[Главный Оркестратор]** пишет Драфт Плана Архитектуры.
2. **[Свита из 3-х Советников]** (Security Advisor, Scale Advisor, Domain Analyst) получает драфт. Им **строго запрещено** общаться между собой. Каждый советник имеет ровно **ОДНУ итерацию (One-Shot)** на ответ. Он возвращает Оркестратору структурированную критику (JSON/Markdown отчет с минусами).
3. **[Главный Оркестратор (Судья)]** читает 3 независимых репорта. Он единолично принимает (или отклоняет) правки и замораживает План. Дискуссий, пересмотров и вторых кругов не существует. Если критика требует полного пересмотра всего концепта — План помечается `BLOCKED` и передается Человеку-заказчику. Бесконечный цикл невозможен математически.

## 🏗️ 2. Штатное Расписание (Механика Глобальных Скиллов)

Все локальные агенты и скрипты из вашего `Everything Claude Code (.agents-global)` остаются фундаментом компании. При создании 100 потоков, Jules MCP не "выдумывает" ИИ заново. Он динамически монтирует нужный скилл из вашей файловой системы в поток.

1. **[Tech Lead]**: Оркеструет процесс.
2. **[Security Auditor]**: `security-auditor` + Sentry MCP.
3. **[QA Engineer]**: `playwright-skill` + Puppeteer MCP.
4. **[Knowledge Oracle]**: Google NotebookLM MCP.
5. **[DevOps/SRE]**: `docker-expert` + `kubernetes-mcp`.
6. **[DBA]**: SQL-миграции + PostgreSQL MCP.
7. **[Frontend Engineer]**: Интеграция с **Google Stitch MCP**.
8. **[Visual AI Artist]**: Генерация графики с **Nanobana MCP**.
9. **[Backend Engineer]**: `tdd-workflow` + **Stitch GraphQL MCP**.

## 📋 3. Воркфлоу: GitHub-First & Contract-Driven

**[Этап -1: GitHub-First]** Любой проект стартует с клонирования/создания репозитория. Внешние инструменты (Google Stitch) пушат сырой код напрямую в ветки GitHub, минуя чат.
**[Этап 0: Совет Оркестраторов]** One-Shot Review плана -> Утверждение OpenAPI Контракта. 
**[Этап 1: Jules Swarm Execution]** Frontend в ветке `feat/front` делает моки, Backend в ветке `feat/api` пишет логику БД.
**[Этап 2: Post-Processing]** Frontend рефакторит код от Stitch, обвязывая его Redux/Zustand локальными стейтами.

## 🛡️ 4. Регламент Отказоустойчивости Роя (Failover vFINAL)

1. **Throttling (5-10 одновременных вызовов)**: Защита от API-банов.
2. **Транзакционная Строгость**: Накат SQL миграций исключительно через `ROLLBACK` оболочку. Перезапись файлов запрещена.
3. **Идемпотентность**: Сброс `git reset --hard` при зависании (Watchdog Kill) агента Jules.
4. **Hard-Limit Бюджет (Circuit Breaker)**: При падении тестов 3 раза подряд — автоматический фриз задачи. Утечка токенов API сведена к нулю.
